Data Strucutres - Graphs

Graphs are data structures that are used to represent relationships between objects. They consist of nodes (vertices) and edges, where the edges connect the nodes. Graphs are used in a variety of applications, including social networks, transportation systems, and computer networks.

Some* Graph Search Algorithms
Depth-First Search (DFS)
DFS is a graph traversal algorithm that explores a graph as far as possible along each branch before backtracking. It uses a stack to keep track of the visited nodes and their neighbors. DFS is useful for exploring all the vertices in a graph and can also be used to detect cycles in a graph. Example: Suppose we have a graph with five nodes and the following edges: (1,2), (1,3), (2,4), (2,5). A DFS traversal of this graph starting from node 1 would visit the nodes in the following order: 1, 2, 4, 5, 3.

Breadth-First Search (DFS)
BFS is another graph traversal algorithm that explores a graph level by level. It uses a queue to keep track of the visited nodes and their neighbors. BFS is useful for finding the shortest path between two nodes in an unweighted graph and can also be used to detect connected components in a graph. Example: Using the same graph as above and starting from node 1, a BFS traversal would visit the nodes in the following order: 1, 2, 3, 4, 5.

A* Algorithm
A* is a pathfinding algorithm that uses a heuristic function to guide its search for the shortest path between two nodes in a weighted graph. It combines the cost of the path from the starting node to the current node with an estimate of the cost of the path from the current node to the destination node. A* is guaranteed to find the optimal path if the heuristic function is admissible and consistent. Example: Suppose we have a weighted graph with five nodes and the following edges and weights: (1,2,3), (1,3,4), (2,4,1), (2,5,2), (3,5,5). If we want to find the shortest path between nodes 1 and 5 using A*, we can use the Manhattan distance between the nodes as the heuristic function. The algorithm would explore the nodes in the following order: 1, 2, 4, 5, and the shortest path would be 1 -> 2 -> 5 with a total weight of 5.

Dijkstra's Algorithm
Dijkstra's algorithm is another pathfinding algorithm that finds the shortest path between two nodes in a weighted graph. It uses a priority queue to keep track of the unvisited nodes and their distances from the starting node. Dijkstra's algorithm is guaranteed to find the optimal path if all the edge weights are non-negative. Example: Using the same weighted graph as above and starting from node 1, Dijkstra's algorithm would explore the nodes in the following order: 1, 2, 4, 5, 3, and the shortest path to node 5 would be 1 -> 2 -> 5 with a total weight of 5.

Greedy Best-First Search
Greedy Best-First Search is a variant of A* that uses only the heuristic function to guide its search. It always expands the node that appears to be closest to the goal based on the heuristic estimate. Greedy BFS is not guaranteed to find the optimal path but can be faster than A* in some cases. Example: Using the same weighted graph as above and starting from node 1, Greedy BFS would explore the nodes in the following order: 1, 2, 5, and the path to node 5 would be 1 -> 2 -> 5 with a total weight of 5.
